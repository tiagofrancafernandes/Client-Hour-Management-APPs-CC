# Tasks

> As tarefas estarão descritas nos arquivos dentro de cada pasta (separada por estapa/status) em '/tasks'.

- `backlog` - Pasta onde ficam as tarefas que ainda não estão programados para serem executados ou podem nem ser executados.
- `paused` - Pasta onde ficam as tarefas pausadas. Pausada para dar prioridade à outra tarefa ou por qualquer razão que seja.
- `doing` - Pasta onde ficam as tarefas em execução.
- `plans` - Pasta onde ficam os planos a serem executados. Podem servir de ponto de partida para as tarefas a serem executadas.
- `todo` - Pasta onde ficam as tarefas definidas, preparadas e com detalhamento faltando apenas serem executadas.
- `done` - Pasta onde ficam as tarefas finalizadas.
- `archive` - Pasta onde ficam as tarefas/planos arquivados. Executados ou não. Nem sempre foram excecutados, às vezes pode simplesmente ter sido deixado de lado mas precisa ser mantido para consulta.

## Fluxo das tarefas

### Fluxo saudável
- planejamento > todo > doing > done

### Fluxo possível
- plan > backlog ... (done|archive)
- plan > todo ... (done|archive)
- (plan|backlog|todo) > doing > done
- (plan|backlog|todo) > todo > doing > done
- (plan|backlog|todo) > todo > doing > paused > doing > done
- (plan|backlog|todo) > todo > doing > paused > doing > done
- (plan|backlog|todo) > doing > paused > doing > done
- (plan|backlog|todo) > archive

## Resumo
> A organização/estrutura serve mais para compreensão visual do que foi feito, será feito ou está sendo feito.
